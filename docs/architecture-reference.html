Data Flow Explanation

Linear Pipeline Architecture (v3.0)

  • Input: User requirement (free text, JIRA, file, or auto-detected)
  • Orchestrator: Parses requirement, creates plan, orchestrates workflow
  • Architect: Designs system topology, APIs, schemas, and UI architecture
  • Implementer: Generates code (detects tech stack), comprehensive tests (95%+), and documentation
  • Skills Applied: Language-specific patterns, frameworks, and standards (Java, Python, React, Spring, etc.)
  • Quality Agent: PR validation, codebase audit, security scan, performance optimization, RCA
  • Tools Support: 25 Python utilities for exporters, code review, analysis, requirements parsing, task generation
  • Business Analyst: JIRA parsing, HTML backlog visualization, reporting
  • Output: Complete deliverables (code + tests + docs + GitHub PR)

Integration Points

  • Skill-Based Architecture: Agents delegate to skills, not monolithic tech-specific agents
  • Linear Pipeline: Explicit handoffs between agents with full context preservation
  • Implementer:Full Function: Runs build + test + doc in same context (no state loss)
  • Tech-Agnostic Agents: Apply language/framework detection at skill application time
  • Cross-Platform Export: Tools export agents & skills to Copilot, Cursor, Claude, etc.
  • Token Optimization: QueryAnalyzer scores requirements before dispatch
  • JIRA Integration: BA agent parses tickets and generates HTML backlog visualizations
