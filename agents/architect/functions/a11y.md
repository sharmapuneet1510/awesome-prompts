# architect:a11y

**Plan accessibility architecture** with WCAG 2.1 compliance, inclusive design, and assistive technology support.

## Inputs

```
architect:a11y requirements="..." [wcag_level="AA"]
```

- `requirements` (string, required) — UI/accessibility requirements
- `wcag_level` (string, optional) — Compliance level (A, AA, AAA)
- `audiences` (string, optional) — Target accessibility needs

## Outputs

```
✓ accessibility-plan.md       — WCAG compliance roadmap
✓ wcag-checklist.md           — Detailed WCAG requirements
✓ inclusive-patterns.md       — Accessible component patterns
✓ testing-guide.md            — Accessibility testing procedures
```

## Example

```bash
architect:a11y requirements="E-commerce platform" [wcag_level="AA"]
```

## Workflow

1. Define accessibility requirements
2. Plan WCAG 2.1 compliance (AA or AAA)
3. Design inclusive component patterns
4. Plan keyboard navigation
5. Document screen reader support
6. Create testing procedures
7. Plan remediation timeline
